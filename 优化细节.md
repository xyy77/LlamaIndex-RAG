## 🛠️ 技术实现步骤（基于现有 RAG Chat）

### 第一步：准备法律知识库

**推荐数据源**（公开免费）：

| 法律文本 | 来源 | 格式 | 说明 |
|---------|------|------|------|
| 《民法典》全文 | 官方发布 PDF / 人大网 | PDF/TXT | 1260 条，核心数据 |
| 《劳动合同法》 | 人社部官网 | PDF/TXT | 98 条，高频查询场景 |
| 《最高人民法院指导案例》摘要 | 中国法院网 | TXT | 可选，增强回答深度 |

**数据处理**：
- 使用你现有的 `SimpleDirectoryReader` 加载 PDF/TXT
- 关键：**按“条款”切分**（而非固定 token 大小）
  - 正则可以按 `第[零〇一二三四五六七八九十百千万0-9]+条` 拆分
  - 每个条款单独作为一个 Document，metadata 记录“法律名称 + 条款号 + 章节标题”
- 这样检索时能精确返回“第 X 条”的内容

**代码示例**（新增 `legal_chunker.py`）：
```python
import re
from llama_index.core import Document

def split_by_article(text: str, law_name: str) -> List[Document]:
    pattern = r'(第[零〇一二三四五六七八九十百千万0-9]+条[、\s]?.*?)(?=第[零〇一二三四五六七八九十百千万0-9]+条|$)'
    articles = re.findall(pattern, text, flags=re.DOTALL)
    docs = []
    for article in articles:
        article_num = re.search(r'第(.*?)条', article).group(0)
        docs.append(Document(
            text=article.strip(),
            metadata={"law_name": law_name, "article": article_num}
        ))
    return docs
```

### 第二步：调整 Prompt 为法律助手风格

修改 `demo.py` 中的 `system_prompt`：

```
你是一个专业的法律条文助手，专注于回答基于《民法典》《劳动合同法》等中国法律的问题。
你必须严格依据提供的法律条款内容回答，不得编造任何法律条文。
如果检索到的条款不能完全回答用户问题，请明确说明“根据现有条款，无法完全回答，建议咨询专业律师”。
回答时请标明所引用的具体条款（例如“依据《民法典》第XX条”）。
最后必须加上免责声明：“以上内容由AI生成，不构成正式法律意见，如有疑问请咨询执业律师。”
```

### 第三步：保留并强化检索策略

- **双索引**：VectorIndex 用于条款细节检索，SummaryIndex 用于法律章节概括
- **Reranker**：开启 gte-rerank（法律术语精确匹配很重要）
- **增加一个后处理**：强制要求 LLM 输出中必须包含条款编号，否则拒绝回答（可设阈值）

### 第四步：UI 增强（展示差异化）

在 Streamlit 界面上增加：

1. **引用溯源区**：在回答下方折叠区域展示“参考条款原文”
2. **条款对比模块**：用户输入“比较《民法典》和《合同法》关于合同无效的规定”，系统分别检索两个法律，用 LLM 对比输出
3. **免责声明**：始终显示在侧边栏或回答底部

### 第五步：评估数据集构造（20-30 条）

从常见法律咨询中抽取，例如：

| 问题 | 参考答案（包含条款） |
|------|---------------------|
| 租房合同未到期，房东能否赶人？ | 《民法典》第725条：租赁物在承租人按照租赁合同占有期限内发生所有权变动的，不影响租赁合同的效力。 |
| 试用期最长多久？ | 《劳动合同法》第19条：劳动合同期限三个月以上不满一年的，试用期不得超过一个月；一年以上不满三年的，试用期不得超过二个月；三年以上固定期限和无固定期限的劳动合同，试用期不得超过六个月。 |

用你的评估脚本计算忠实度和相关性（目标 90%+）。
